TrustVault Blockchain API Integration — Credentials & Auth Spec

Base URL: https://dwtl.io

Registered App ID: dw_app_trustvault

Credentials (set these as environment variables on the TrustVault side):

The two values you need are stored as secrets in the dwtl.io workspace under:

TRUSTLAYER_API_KEY → use as BLOCKCHAIN_API_KEY
TRUSTLAYER_API_SECRET → use as BLOCKCHAIN_API_SECRET
Retrieve the actual values from the dwtl.io Secrets tab and copy them to TrustVault's environment.

Authentication: HMAC-SHA256

Every request must include three headers:

Header	Value
x-blockchain-key	Your BLOCKCHAIN_API_KEY
x-blockchain-signature	HMAC-SHA256 signature (see below)
x-blockchain-timestamp	Unix timestamp in milliseconds (e.g. Date.now())
Signature Generation (CRITICAL — must match exactly):

Canonical string = METHOD:PATH:API_KEY:TIMESTAMP:BODY_HASH
Where:
  METHOD    = HTTP method uppercase (GET, POST)
  PATH      = Request path (e.g. /api/identity/anchor)
  API_KEY   = Value of x-blockchain-key header
  TIMESTAMP = Value of x-blockchain-timestamp header
  BODY_HASH = SHA-256 hex of JSON.stringify(requestBody) for POST
              Empty string "" for GET requests or empty bodies
Then sign: HMAC-SHA256(BLOCKCHAIN_API_SECRET, canonicalString) → hex digest

Timestamp window: Must be within 5 minutes of server time or request is rejected.

Node.js Reference Implementation:

const crypto = require("crypto");
function signRequest(method, path, body = {}) {
  const apiKey = process.env.BLOCKCHAIN_API_KEY;
  const apiSecret = process.env.BLOCKCHAIN_API_SECRET;
  const timestamp = Date.now().toString();
  const bodyStr = JSON.stringify(body);
  const bodyHash = method === "GET" || Object.keys(body).length === 0
    ? ""
    : crypto.createHash("sha256").update(bodyStr).digest("hex");
  const canonical = `${method}:${path}:${apiKey}:${timestamp}:${bodyHash}`;
  const signature = crypto.createHmac("sha256", apiSecret).update(canonical).digest("hex");
  return {
    "x-blockchain-key": apiKey,
    "x-blockchain-signature": signature,
    "x-blockchain-timestamp": timestamp,
  };
}
Available Endpoints:

Method	Path	Auth	Description
POST	/api/identity/anchor	HMAC	Anchor Trust Layer ID on-chain
GET	/api/identity/verify/{trustLayerId}	None	Verify on-chain identity
GET	/api/identity/resolve/{trustLayerId}	None	Resolve identity details
POST	/api/provenance/register	HMAC	Register media file provenance
GET	/api/provenance/verify/{provenanceId}	None	Verify provenance record
GET	/api/trust/score/{trustLayerId}	None	Query trust score
GET	/api/trust/relationship/{idA}/{idB}	None	Query trust relationship
POST	/api/trust/verify	HMAC	Verify trust assertion
GET	/api/signal/balance/{trustLayerId}	None	Query SIG balance
POST	/api/signal/transfer	HMAC	Transfer SIG tokens
POST	/api/signal/gate	HMAC	Check Signal-gated access
Note: GET endpoints are public (no HMAC needed). POST endpoints require full HMAC authentication.

Error Codes:

Code	Meaning
AUTH_MISSING	Missing one or more auth headers
AUTH_INVALID	API key doesn't match
SIG_INVALID	Signature verification failed
TIMESTAMP_INVALID	Timestamp expired or invalid
CONFIG_ERROR	Server-side API not configured (503)
